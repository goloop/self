# 05. Ask a language model, swap the provider

**Task.** Call a large language model from Go, and keep the calling code the
same whether you use Anthropic or OpenAI. Swapping the provider should be a one
line change, not a rewrite.

**Modules.** [`ai`](https://github.com/goloop/ai) is the provider-agnostic
contract: `Client`, `Request`, `Response`, `Chunk`. Each driver implements it -
[`anthropic`](https://github.com/goloop/anthropic) and
[`openai`](https://github.com/goloop/openai) here, with the same interface for
the others. Your code speaks `ai.Client`; only the constructor and the model
name change.

**Recipe.** [`recipes/005-ai`](../recipes/005-ai/)

## The idea

`ai` is to language models what `database/sql` is to databases: one small
interface, many drivers. You build a request once and hand it to any `Client`:

```go
var client ai.Client
client = anthropic.New(anthropicKey) // or:
client = openai.New(openaiKey)       // same interface

resp, err := client.Generate(ctx, &ai.Request{
	Model:     model, // the only provider-specific value
	System:    "You are terse. Answer in one short sentence.",
	Messages:  []ai.Message{ai.UserText("What is a database index?")},
	MaxTokens: 80,
})
fmt.Println(resp.Text(), resp.Usage.InputTokens, resp.Usage.OutputTokens)
```

A provider is used only when its key is configured, so the same program runs
with one provider or both. No key is stored in the repository.

## Example A - one question, every provider

The recipe collects the configured providers and asks each the same question.
The code inside the loop is identical; only `Model` differs:

```go
for _, p := range configured() {
	resp, err := p.client.Generate(ctx, &ai.Request{
		Model: p.model, System: "You are terse. Answer in one short sentence.",
		Messages: []ai.Message{ai.UserText("What is a database index?")}, MaxTokens: 80,
	})
	// ... print resp.Text() and resp.Usage
}
```

## Example B - streaming

`Client.Stream` returns an iterator of `Chunk` values; the text arrives as it is
produced, and the final chunk carries the token usage:

```go
for chunk, err := range p.client.Stream(ctx, req) {
	if err != nil {
		break
	}
	fmt.Print(chunk.Text)
	if chunk.Done {
		fmt.Printf(" (out=%d)", chunk.Usage.OutputTokens)
	}
}
```

## Example C - shaping the answer

The `Request` fields are your controls. A `System` prompt sets the behavior, and
a small `MaxTokens` caps the length - here, forcing a single word:

```go
resp, _ := p.client.Generate(ctx, &ai.Request{
	Model: p.model, System: "Reply with exactly one word and nothing else.",
	Messages: []ai.Message{ai.UserText("The capital of France?")}, MaxTokens: 5,
})
// resp.Text() == "Paris"
```

Tested with one cheap call per provider, then run against the real APIs:

```text
$ go test ./...            # one call per key; skips when no key is set

$ go run .
A. one question, every provider (Generate):
   [anthropic] in=24 out=29: A database index is a data structure that speeds up queries by allowing the database to find data without scanning every row in a table.
   [openai] in=27 out=20: A database index is a data structure that improves the speed of data retrieval operations on a database table.
B. streaming from anthropic (Stream):
   fmt, os, net
   (done, out=8 tokens)

   -> "Paris"
```

Both providers answered example A through the identical `Generate` call and
reported their token usage; the stream in B printed the answer as it arrived;
and the `System` prompt in C constrained the reply to one word.

## What you learned

- `ai.Client` is one interface with many drivers, the way `database/sql` is.
  Write against `ai.Request`/`Response`; swap `anthropic.New` for `openai.New`.
- `Generate` returns a whole `Response` (with `Text()` and `Usage`);
  `Stream` returns an iterator of `Chunk` for incremental output.
- The `Request` fields (`System`, `MaxTokens`, `Messages`, ...) are the
  controls; only `Model` is provider-specific.
- Enable a provider only when its key is present, so the program degrades to
  whatever is configured. Never commit an API key.
