# 05. Ask a language model - text, photos and tools

**Task.** Call a large language model from Go and keep the calling code the same
across providers. Send it text, stream the reply, send it a **photo**, and let
it call your own **functions** - all through one interface.

**Modules.** [`ai`](https://github.com/goloop/ai) is the provider-agnostic
contract: `Client`, `Request`, `Response`, `Chunk`, and the content parts
`Text`, `Image`, `ToolUse`, `ToolResult`. Each driver implements it -
[`anthropic`](https://github.com/goloop/anthropic) and
[`openai`](https://github.com/goloop/openai) here. Your code speaks `ai.Client`;
only the constructor and the model name change.

**Recipe.** [`recipes/005-ai`](../recipes/005-ai/)

## Construct a client

`anthropic.New(key)` and `openai.New(key)` both return an `ai.Client`. That is
the only provider-specific line in the whole program:

```go
var client ai.Client
client = anthropic.New(os.Getenv("ANTHROPIC_API_KEY")) // model: anthropic.ModelClaudeHaiku45
client = openai.New(os.Getenv("OPENAI_API_KEY"))       // model: openai.ModelGPT4oMini
```

A provider is used only when its key is set, so the same program runs with one
or both. No key is stored in the repository.

## Example A - ask a question

`Generate` sends a `Request` and returns a whole `Response`. `Text()` is the
answer; `Usage` is the token count. The same call works on either client:

```go
resp, err := client.Generate(ctx, &ai.Request{
	Model:     model,
	System:    "You are terse. Answer in one short sentence.",
	Messages:  []ai.Message{ai.UserText("What is a database index?")},
	MaxTokens: 80,
})
fmt.Println(resp.Text(), resp.Usage.InputTokens, resp.Usage.OutputTokens)
```

## Example B - stream the reply

`Stream` returns an iterator of `Chunk`; the text arrives as it is produced, and
the final chunk carries the usage:

```go
for chunk, err := range client.Stream(ctx, req) {
	if err != nil {
		break
	}
	fmt.Print(chunk.Text)
	if chunk.Done {
		fmt.Printf(" (out=%d)", chunk.Usage.OutputTokens)
	}
}
```

## Example C - shape the reply

The `Request` fields are your controls. A `System` prompt sets the behavior; a
small `MaxTokens` caps the length - here, forcing one word:

```go
resp, _ := client.Generate(ctx, &ai.Request{
	Model: model, System: "Reply with exactly one word and nothing else.",
	Messages: []ai.Message{ai.UserText("The capital of France?")}, MaxTokens: 5,
})
// resp.Text() == "Paris"
```

## Example D - send a photo

A `Message` can mix parts. To send an image, put an `ai.Image` next to a
`ai.Text` part; the image carries a MIME type and either inline `Data` bytes or
a `URL`. Here the bytes are a small PNG generated in memory:

```go
resp, _ := client.Generate(ctx, &ai.Request{
	Model: model, MaxTokens: 20,
	Messages: []ai.Message{{
		Role: ai.RoleUser,
		Parts: []ai.Part{
			ai.Text{Text: "What color is this square? Answer with one word."},
			ai.Image{MIME: "image/png", Data: pngBytes}, // or ai.Image{URL: "https://..."}
		},
	}},
})
// resp.Text() == "Red"
```

## Example E - let the model call a tool

Describe a function with `ai.Tool` (a name, a description and a JSON Schema),
and the model can ask to call it. You run the function and hand the result back
with an `ai.ToolResult`; the model then turns the data into an answer:

```go
weather := ai.Tool{
	Name: "get_weather", Description: "Get the current weather for a city",
	Schema: json.RawMessage(`{"type":"object","properties":{"city":{"type":"string"}},"required":["city"]}`),
}

// 1) the model asks to call the tool:
first, _ := client.Generate(ctx, &ai.Request{
	Model: model, Tools: []ai.Tool{weather}, ToolChoice: ai.ToolAuto,
	Messages: []ai.Message{ai.UserText("What is the weather in Kyiv? Use the tool.")},
})
call := first.ToolCalls()[0] // call.Name == "get_weather", call.Input == {"city":"Kyiv"}

// 2) you answer the call and ask again:
final, _ := client.Generate(ctx, &ai.Request{
	Model: model, Tools: []ai.Tool{weather},
	Messages: []ai.Message{
		ai.UserText("What is the weather in Kyiv? Use the tool."),
		{Role: ai.RoleAssistant, Parts: []ai.Part{call}},
		{Role: ai.RoleTool, Parts: []ai.Part{ai.ToolResult{ID: call.ID, Content: `{"temp_c":15,"sky":"sunny"}`}}},
	},
})
// final.Text() describes the weather in prose
```

## What you learned

- `anthropic.New` and `openai.New` both return an `ai.Client`; write against
  `ai.Request`/`Response` and only the model name is provider-specific.
- `Generate` returns the whole `Response`; `Stream` returns a `Chunk` iterator;
  `System` and `MaxTokens` shape the reply.
- A `Message` holds a list of `Part`s: put an `ai.Image` (inline `Data` or a
  `URL`) next to a `ai.Text` to send a photo.
- `ai.Tool` describes a function; the model returns a `ToolUse` (name + JSON
  arguments), you answer with a `ToolResult`, and it produces the final answer.
- Enable a provider only when its key is present. Never commit an API key.
