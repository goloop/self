# 05. Запитати мовну модель - текст, фото й інструменти

**Задача.** Викликати велику мовну модель з Go так, щоб код виклику лишався тим
самим на різних провайдерах. Надіслати текст, стрімити відповідь, надіслати
**фото** і дати їй викликати ваші власні **функції** - усе через один інтерфейс.

**Модулі.** [`ai`](https://github.com/goloop/ai) - провайдер-незалежний контракт:
`Client`, `Request`, `Response`, `Chunk` і частини вмісту `Text`, `Image`,
`ToolUse`, `ToolResult`. Кожен драйвер його реалізує - тут
[`anthropic`](https://github.com/goloop/anthropic) і
[`openai`](https://github.com/goloop/openai). Ваш код говорить `ai.Client`;
змінюються лише конструктор і назва моделі.

**Рецепт.** [`recipes/005-ai`](../recipes/005-ai/)

## Створіть клієнт

`anthropic.New(key)` і `openai.New(key)` обидва повертають `ai.Client`. Це єдиний
провайдер-специфічний рядок у всій програмі:

```go
var client ai.Client
client = anthropic.New(os.Getenv("ANTHROPIC_API_KEY")) // модель: anthropic.ModelClaudeHaiku45
client = openai.New(os.Getenv("OPENAI_API_KEY"))       // модель: openai.ModelGPT4oMini
```

Провайдер використовується лише тоді, коли задано його ключ, тож та сама
програма працює з одним або обома. Жоден ключ не зберігається в репозиторії.

## Приклад A - поставте питання

`Generate` надсилає `Request` і повертає цілу `Response`. `Text()` - відповідь;
`Usage` - облік токенів. Той самий виклик працює на будь-якому клієнті:

```go
resp, err := client.Generate(ctx, &ai.Request{
	Model:     model,
	System:    "You are terse. Answer in one short sentence.",
	Messages:  []ai.Message{ai.UserText("What is a database index?")},
	MaxTokens: 80,
})
fmt.Println(resp.Text(), resp.Usage.InputTokens, resp.Usage.OutputTokens)
```

## Приклад B - стрімте відповідь

`Stream` повертає ітератор `Chunk`; текст надходить у міру генерації, а фінальний
чанк несе облік:

```go
for chunk, err := range client.Stream(ctx, req) {
	if err != nil {
		break
	}
	fmt.Print(chunk.Text)
	if chunk.Done {
		fmt.Printf(" (out=%d)", chunk.Usage.OutputTokens)
	}
}
```

## Приклад C - надайте форму відповіді

Поля `Request` - це ваші ручки. `System`-промпт задає поведінку; малий
`MaxTokens` обмежує довжину - тут змушуючи одне слово:

```go
resp, _ := client.Generate(ctx, &ai.Request{
	Model: model, System: "Reply with exactly one word and nothing else.",
	Messages: []ai.Message{ai.UserText("The capital of France?")}, MaxTokens: 5,
})
// resp.Text() == "Paris"
```

## Приклад D - надішліть фото

`Message` може змішувати частини. Щоб надіслати зображення, поставте `ai.Image`
поруч із частиною `ai.Text`; зображення несе MIME-тип і або інлайн-байти `Data`,
або `URL`. Тут байти - маленький PNG, згенерований у пам'яті:

```go
resp, _ := client.Generate(ctx, &ai.Request{
	Model: model, MaxTokens: 20,
	Messages: []ai.Message{{
		Role: ai.RoleUser,
		Parts: []ai.Part{
			ai.Text{Text: "What color is this square? Answer with one word."},
			ai.Image{MIME: "image/png", Data: pngBytes}, // або ai.Image{URL: "https://..."}
		},
	}},
})
// resp.Text() == "Red"
```

## Приклад E - дайте моделі викликати інструмент

Опишіть функцію через `ai.Tool` (ім'я, опис і JSON Schema), і модель зможе
попросити її викликати. Ви виконуєте функцію й повертаєте результат через
`ai.ToolResult`; далі модель перетворює дані на відповідь:

```go
weather := ai.Tool{
	Name: "get_weather", Description: "Get the current weather for a city",
	Schema: json.RawMessage(`{"type":"object","properties":{"city":{"type":"string"}},"required":["city"]}`),
}

// 1) модель просить викликати інструмент:
first, _ := client.Generate(ctx, &ai.Request{
	Model: model, Tools: []ai.Tool{weather}, ToolChoice: ai.ToolAuto,
	Messages: []ai.Message{ai.UserText("What is the weather in Kyiv? Use the tool.")},
})
call := first.ToolCalls()[0] // call.Name == "get_weather", call.Input == {"city":"Kyiv"}

// 2) ви відповідаєте на виклик і питаєте знову:
final, _ := client.Generate(ctx, &ai.Request{
	Model: model, Tools: []ai.Tool{weather},
	Messages: []ai.Message{
		ai.UserText("What is the weather in Kyiv? Use the tool."),
		{Role: ai.RoleAssistant, Parts: []ai.Part{call}},
		{Role: ai.RoleTool, Parts: []ai.Part{ai.ToolResult{ID: call.ID, Content: `{"temp_c":15,"sky":"sunny"}`}}},
	},
})
// final.Text() описує погоду прозою
```

## Звіт виконання

Протестовано (дешевий виклик на провайдера плюс vision-виклик), потім запущено
проти справжніх API:

```text
$ go test ./...          # generate + vision; без ключа - пропускає
ok  	goloop.one/handbook/005-ai	2.506s

$ go run .
A. construct + ask (anthropic.New / openai.New, Generate):
   [anthropic] in=24 out=34: A database index is a data structure that speeds up data retrieval by creating a sorted reference to rows in a table, similar to a book's index.
   [openai] in=27 out=20: A database index is a data structure that improves the speed of data retrieval operations on a database table.
B. stream from anthropic:
   fmt, os, net
   (done, out=8 tokens)
C. shape with System + MaxTokens (anthropic):
   -> "Paris"
D. photo / vision (anthropic):
   sent a 137-byte PNG, model saw: "Red"
E. tools / function calling (anthropic):
   model called get_weather({"city":"Kyiv"})
   after the tool result: The weather in Kyiv is currently 15C and sunny.
```

Обидва провайдери відповіли на A через ідентичний `Generate`; B стрімив; C
обмежено одним словом; **D** надіслав справжні байти зображення, і модель
прочитала колір; а **E** показує повний цикл інструмента - модель попросила
`get_weather`, ви повернули дані, і вона написала фінальне речення.

## Що ви дізналися

- `anthropic.New` і `openai.New` обидва повертають `ai.Client`; пишіть проти
  `ai.Request`/`Response`, і лише назва моделі провайдер-специфічна.
- `Generate` повертає цілу `Response`; `Stream` повертає ітератор `Chunk`;
  `System` і `MaxTokens` надають форму відповіді.
- `Message` тримає список `Part`: поставте `ai.Image` (інлайн `Data` чи `URL`)
  поруч із `ai.Text`, щоб надіслати фото.
- `ai.Tool` описує функцію; модель повертає `ToolUse` (ім'я + JSON-аргументи), ви
  відповідаєте `ToolResult`, і вона видає фінальну відповідь.
- Вмикайте провайдера лише тоді, коли є його ключ. Ніколи не комітьте API-ключ.
