# 05. Запитати мовну модель, змінити провайдера

**Задача.** Викликати велику мовну модель з Go так, щоб код виклику лишався тим
самим, чи ви берете Anthropic, чи OpenAI. Зміна провайдера має бути правкою в
один рядок, а не переписуванням.

**Модулі.** [`ai`](https://github.com/goloop/ai) - це провайдер-незалежний
контракт: `Client`, `Request`, `Response`, `Chunk`. Кожен драйвер його реалізує -
тут [`anthropic`](https://github.com/goloop/anthropic) і
[`openai`](https://github.com/goloop/openai), з тим самим інтерфейсом для решти.
Ваш код говорить `ai.Client`; змінюються лише конструктор і назва моделі.

**Рецепт.** [`recipes/005-ai`](../recipes/005-ai/)

## Ідея

`ai` для мовних моделей - те саме, що `database/sql` для баз: один малий
інтерфейс, багато драйверів. Ви будуєте запит один раз і віддаєте його будь-якому
`Client`:

```go
var client ai.Client
client = anthropic.New(anthropicKey) // або:
client = openai.New(openaiKey)       // той самий інтерфейс

resp, err := client.Generate(ctx, &ai.Request{
	Model:     model, // єдине провайдер-специфічне значення
	System:    "You are terse. Answer in one short sentence.",
	Messages:  []ai.Message{ai.UserText("What is a database index?")},
	MaxTokens: 80,
})
fmt.Println(resp.Text(), resp.Usage.InputTokens, resp.Usage.OutputTokens)
```

Провайдер використовується лише тоді, коли задано його ключ, тож та сама
програма працює з одним провайдером або з обома. Жоден ключ не зберігається в
репозиторії.

## Приклад A - одне питання, кожен провайдер

Рецепт збирає налаштовані провайдери й ставить кожному те саме питання. Код
усередині циклу однаковий; змінюється лише `Model`:

```go
for _, p := range configured() {
	resp, err := p.client.Generate(ctx, &ai.Request{
		Model: p.model, System: "You are terse. Answer in one short sentence.",
		Messages: []ai.Message{ai.UserText("What is a database index?")}, MaxTokens: 80,
	})
	// ... надрукувати resp.Text() і resp.Usage
}
```

## Приклад B - стрімінг

`Client.Stream` повертає ітератор значень `Chunk`; текст надходить у міру
генерації, а фінальний чанк несе облік токенів:

```go
for chunk, err := range p.client.Stream(ctx, req) {
	if err != nil {
		break
	}
	fmt.Print(chunk.Text)
	if chunk.Done {
		fmt.Printf(" (out=%d)", chunk.Usage.OutputTokens)
	}
}
```

## Приклад C - надати форму відповіді

Поля `Request` - це ваші ручки. `System`-промпт задає поведінку, а малий
`MaxTokens` обмежує довжину - тут змушуючи одне слово:

```go
resp, _ := p.client.Generate(ctx, &ai.Request{
	Model: p.model, System: "Reply with exactly one word and nothing else.",
	Messages: []ai.Message{ai.UserText("The capital of France?")}, MaxTokens: 5,
})
// resp.Text() == "Paris"
```

## Звіт виконання

Протестовано одним дешевим викликом на провайдера, потім запущено проти
справжніх API:

```text
$ go test ./...            # один виклик на ключ; без ключа - пропускає
ok  	goloop.one/handbook/005-ai	3.449s

$ go run .
A. one question, every provider (Generate):
   [anthropic] in=24 out=29: A database index is a data structure that speeds up queries by allowing the database to find data without scanning every row in a table.
   [openai] in=27 out=20: A database index is a data structure that improves the speed of data retrieval operations on a database table.
B. streaming from anthropic (Stream):
   fmt, os, net
   (done, out=8 tokens)
C. shaping the answer with System + MaxTokens (anthropic):
   -> "Paris"
```

Обидва провайдери відповіли на приклад A через ідентичний виклик `Generate` і
повідомили облік токенів; стрім у B надрукував відповідь у міру надходження; а
`System`-промпт у C обмежив відповідь одним словом.

## Що ви дізналися

- `ai.Client` - це один інтерфейс із багатьма драйверами, як `database/sql`.
  Пишіть проти `ai.Request`/`Response`; міняйте `anthropic.New` на `openai.New`.
- `Generate` повертає цілу `Response` (з `Text()` і `Usage`); `Stream` повертає
  ітератор `Chunk` для інкрементного виводу.
- Поля `Request` (`System`, `MaxTokens`, `Messages`, ...) - це ручки; лише
  `Model` провайдер-специфічний.
- Вмикайте провайдера лише тоді, коли є його ключ, тож програма деградує до
  того, що налаштовано. Ніколи не комітьте API-ключ.
