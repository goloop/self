# Recipe: 004-postgresql

Part of the [GoLoop One handbook](../../main.md). Read the chapter:
[04-postgresql](../../chapters/04-postgresql.md).

Run it against a real database:

```sh
docker compose up -d                 # a throwaway PostgreSQL on :5440
export PGC_DATABASE_URL='postgres://book:secret@127.0.0.1:5440/book?sslmode=disable'

pgc migrate                          # apply migrations/
pgc generate                         # regenerate internal/store/ (already committed)
go test ./...                        # runs against the DB; skips when it is unset
go run .
```

`internal/store/` is generated by `pgc generate`; it is committed so the recipe
runs without regenerating.
