# GoLoop One - книга рецептів для Go API

📖 [English](main.md) · **Українська**

*Одне рішення для Go API, задача за задачею.*

Вітаємо. Ця книга допоможе вам збудувати Go API за допомогою GoLoop, починаючи з
того, що вам справді треба зробити: прочитати конфігурацію, віддати JSON по HTTP,
почистити форму реєстрації, зберегти рядки в PostgreSQL, запитати мовну модель.
На кожну задачу ви отримуєте рецепт, який можна прочитати, запустити й
пристосувати.

Кожен рецепт - це повноцінна програма в теці [`recipes/`](recipes/), зі своїм
`go.mod`, `main.go` і `main_test.go`. Глава показує реальний вивід її запуску,
тож ви бачите саме те, що код робить.

## Як користуватися книгою

- Читайте главу заради ідеї; відкривайте теку рецепта заради цілої програми.
- Щоб запустити рецепт самостійно:

  ```sh
  cd recipes/<назва>
  go test ./...
  go run .
  ```

- Глави впорядковані, але незалежні. Частина I бере по одному модулю за раз;
  подальші частини поєднують їх у сервіс.
- Ви берете один рецепт, не беручи решту: кожен модуль GoLoop - окремий
  [stdlib-first](../README.UK.md) пакет зі своєю версією.

## Зміст

- [02. JSON HTTP API без фреймворка](chapters/02-http-json-api.uk.md) -

  `is`, `norm` · рецепт [`003-validate-and-clean`](recipes/003-validate-and-clean/)

### Частина II - дані й моделі *(у роботі)*

- 04. Типобезпечний PostgreSQL з міграціями - `pgc`
- 05. Запитати мовну модель, змінити провайдера - `ai`, `anthropic`, `openai`
- 06. Сесії, токени й паролі - `auth`, `jwt`, `session`, `key`
- 07. Слаги, транслітерація й регістри - `slug`, `t13n`, `scs`

### Частина III - увесь стек *(у планах)*

- 08. Життєвий цикл сервісу - `app`, `observe`, `log`
- 09. Реальний час на WebSocket - `websocket`
- 10. Складаємо все разом - один API на цілому стеку GoLoop
