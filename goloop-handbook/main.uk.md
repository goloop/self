# GoLoop One - книга рецептів для Go API

📖 [English](main.md) · **Українська**

*Одне рішення для Go API, задача за задачею.*

Це книга, а не довідник. Довідник розповідає, що робить кожна функція; ця книга
починається з задачі, яка справді є в людини - «прочитати конфігурацію»,
«віддати JSON по HTTP», «почистити форму реєстрації», «зберегти рядки в
PostgreSQL», «запитати модель» - і показує робочий рецепт, який розв'язує її за
допомогою GoLoop.

Кожен рецепт у книзі - це повноцінна робоча Go-програма в теці
[`recipes/`](recipes/). Кожну спершу збирають, тестують і запускають, і лише
потім про неї пишуть, а справжній вивід того запуску друкують у главі як **звіт
виконання**. Коли рецепту потрібна база чи мовна модель, звіт отримано на
справжній. Тут немає псевдокоду.

## Як читати цю книгу

- Читай главу заради ідеї; відкривай теку рецепта заради цілої програми.
- У кожній теці рецепта є `go.mod`, `main.go` і `main_test.go`. Щоб запустити
  самому: `cd recipes/<назва> && go test ./... && go run .`
- Глави впорядковані, але незалежні. Кожна починається з основ і нарощує.
  Остання частина збирає окремі шматки в один сервіс.
- Кожен модуль GoLoop [спирається на stdlib і майже без залежностей](../README.UK.md):
  ти береш один рецепт, не беручи решту.

## Зміст

### Частина I - по одному модулю за раз

Кожна глава розв'язує одну звичайну задачу найменшим набором модулів.

- [01. Конфігурація, якою можна керувати](chapters/01-configuration.uk.md) -
  `env`, `opt` · рецепт [`001-configuration`](recipes/001-configuration/)
- [02. JSON HTTP API без фреймворка](chapters/02-http-json-api.uk.md) -
  `mux`, `resp`, `middlewares` · рецепт [`002-http-json-api`](recipes/002-http-json-api/)
- [03. Валідація й чистка вводу](chapters/03-validate-and-clean.uk.md) -
  `is`, `norm` · рецепт [`003-validate-and-clean`](recipes/003-validate-and-clean/)

### Частина II - дані й моделі *(у планах)*

- 04. Типобезпечний PostgreSQL з міграціями - `pgc`
- 05. Запитати мовну модель, змінити провайдера - `ai`, `anthropic`, `openai`
- 06. Сесії, токени й паролі - `auth`, `jwt`, `session`, `key`
- 07. Слаги, транслітерація й регістри - `slug`, `t13n`, `scs`

### Частина III - увесь стек *(у планах)*

- 08. Життєвий цикл сервісу - `app`, `observe`, `log`
- 09. Реальний час на WebSocket - `websocket`
- 10. Складаємо все разом - один API на цілому стеку GoLoop

Частину I написано й перевірено. Частини II і III у дорозі; рецепти під них
будують і тестують так само, по одному.
